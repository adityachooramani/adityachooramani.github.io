1. Geospatial Processing Pipeline#

The engine uses GDAL, Rasterio, and NumPy to perform multi-index change detection:

  • Deforestation Detection: NDVI thresholding (Drop > 0.3).
  • Urban Expansion: NDBI rise monitoring (Rise > 0.25).
  • Encroachment Detection: Vectorizing raster data and performing spatial intersections with protected area boundaries via GeoPandas.

2. Solving Hardware Constraints#

Processing 400MB+ GeoTIFFs on an EC2 instance with limited RAM (4GB) required engineering workarounds:

  • Tiled Processing: Reading rasters in 1024x1024 pixel chunks instead of loading the entire file into memory.
  • Memory Swap: Configured a 4GB swap file on the EBS volume to prevent OOM (Out of Memory) crashes.

3. Security Implementation#

Following the Principle of Least Privilege, I implemented:

  • S3 Presigned URLs: Results are shared via time-limited (1-hour) links, keeping the S3 bucket private.
  • IAM Granularity: Lambda roles are restricted to specific bucket resources and DynamoDB GSI indexes.
  • Encryption: AES-256 server-side encryption enabled for data at rest in S3 and DynamoDB.
